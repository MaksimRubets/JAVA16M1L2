1) git init -> ініціалізація git репозиторії
2) git status - > відобразити статус файлів в репозиторії
3) git add filename -> додати файл в staging area
4) git commit -m "commit msg" -> зробити commit
5) git log -> відобразити log комітів
6) git diff Head^ Head -> перегляд змін між комітами

7) git remote add origin git@github.com:MaksimRubets/JAVA16M1L2.git -> 
	git remote add -> ця команда створює посилання на віддалений репозиторій
	origin -> це стандартна назва (аліас) для першого та основного віддаленого репозиторію
	git@github.com:... -> це URL-адреса твого репозиторію на GitHub, яка використовує протокол SSH (Secure Shell)

8) git remote -v -> перевіряє список віддалених репозиторіїв та їхні URL-адреси
	Вивід: Підтверджує, що аліас origin налаштований як для отримання (fetch), так і для надсилання (push) даних

9) git push -u origin main ->
	git push: Надсилає твої локальні фіксації на віддалений сервер
	-u (або --set-upstream): Цей прапорець встановлює зв'язок між твоєю локальною гілкою main і віддаленою гілкою origin/main. Це означає, що наступного разу ти зможеш просто використовувати git push (без вказівки origin main)
	Результат:
	Git підрахував твої об'єкти (objects: 24), стиснув їх і відправив.
	Рядок * [new branch] main -> main підтверджує, що локальна гілка main була створена як віддалена гілка main на GitHub.
	branch 'main' set up to track 'origin/main'. підтверджує встановлення зв'язку

10) git pull ->
	Команда git pull виконує дві основні дії послідовно:
	git fetch -> Завантажує останні зміни (нові фіксації) з віддаленого репозиторію (GitHub) у твій локальний Git-репозиторій.
	git merge -> Автоматично зливає ці завантажені зміни у твою поточну робочу гілку (main).
	Переконайся, що ти знаходишся у кореневій папці твого проєкту (java-lect-16-github), а потім виконай:
	git pull origin main
	Або, оскільки ти вже налаштував відстеження гілки, ти можеш просто ввести:
	git pull
